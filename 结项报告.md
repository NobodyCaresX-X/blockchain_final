# 区块链众筹系统 - 结项报告

## 1. 项目概述

本项目是一个基于区块链技术的去中心化众筹平台，采用以太坊智能合约实现资金的安全管理和透明化。系统支持用户创建众筹项目、捐赠资金、阶段性里程碑释放资金以及退款等核心功能。

### 1.1 项目目标

- 构建安全、透明、可追溯的众筹平台
- 实现基于智能合约的资金自动管理
- 提供阶段点机制，支持阶段性资金释放
- 提供里程碑机制，发起者承诺回报内容
- 支持月度支持功能，实现长期捐赠
- 支持用户认证、权限管理和交易记录

### 1.2 核心特性

- ✅ 去中心化：基于以太坊智能合约，资金由合约自动管理
- ✅ 透明可追溯：所有交易记录在区块链上公开可查
- ✅ 双机制设计：**阶段点**（资金分批释放）与**里程碑**（承诺回报）分离
- ✅ 月度支持：支持按月自动扣款的长期捐赠模式
- ✅ 自动结束：项目到期后自动结束，无需创建者确认
- ✅ 无里程碑项目达标立即结束：无里程碑的众筹达到目标后立即结束，溢出金额退还最后捐赠者
- ✅ 资金自动到账：成功项目阶段点默认已完成，一次性全额打款给发起者
- ✅ 比例退款：众筹失败时按剩余资金比例退款，保留原始捐赠记录（使用donatedRefunded映射标记）
- ✅ 早期支持者奖励：前10名捐赠者获得早期支持者标识
- ✅ 时间同步：链上时间与系统时间自动同步，误差≤1分钟，支持UTC+8时区显示
- ✅ 阶段点自动保留：即使发起者未确认阶段点完成，已达标阶段点资金自动保留给发起者
- ✅ 失败项目显示原始金额：捐赠者金额显示原始捐赠金额，不按退款比例计算
- ✅ 前端截止时间自动调整：链上时间超前时自动调整截止时间确保有效性

## 3. 核心概念说明

### 3.1 阶段点（Stage）与里程碑（Reward）的区别

| 特性 | 阶段点（Stage） | 里程碑（Reward） |
|------|----------------|-----------------|
| **用途** | 达到目标比例后释放资金（分批打款） | 发起者承诺的回报内容（超产目标） |
| **设置项** | 达标比例 + 释放比例 | 达标金额（超目标）+ 承诺内容 + 预计实现时间 |
| **触发条件** | 筹集金额达到目标比例 | 筹集金额达到达标金额（必须超过目标） |
| **效果** | 释放对应比例的**目标金额**给创建者 | 仅展示承诺信息，无资金释放 |
| **退款影响** | 已达标阶段点资金自动保留，剩余按比例退 | 无直接影响 |
| **自动处理** | 项目失败时自动计算已达标阶段点并保留资金 | 更新支持者数量 |
| **达标影响** | 项目成功后所有阶段点默认已完成，一次性全额打款 | 无影响 |
| **项目结束** | 不影响项目是否自动结束 | 有里程碑时项目达到目标后继续运行，无里程碑时立即结束 |

### 3.2 项目成功标准

- **项目成功**：筹集金额 **≥ 目标金额（100%）**
- **项目失败**：筹集金额 **< 目标金额（100%）**
- **无里程碑项目**：达到目标后立即结束，溢出金额退还最后捐赠者，资金自动全额打款给发起者
- **有里程碑项目**：达到目标后继续运行，直到截止时间，期间可继续接受捐赠
- **成功项目处理**：所有阶段点自动标记为已完成/已释放，一次性全额打款给发起者
- **失败项目处理**：按比例退款，已达标阶段点资金保留给发起者，捐赠者金额显示原始金额

### 3.3 阶段点资金释放规则

阶段点资金释放基于**目标金额**而非实际筹集金额：

| 场景 | 阶段点达标 | 释放金额计算 |
|------|-----------|-------------|
| 目标100ETH，阶段点60%/20% | 已达标 | 100ETH × 20% = 20ETH |
| 目标100ETH，阶段点80%/40% | 已达标 | 100ETH × 40% = 40ETH |
| 实际筹集90ETH（失败） | - | 总保留 = 60ETH，可退款30ETH |
| 实际筹集100+ETH（成功） | - | 所有阶段点默认已完成，一次性全额打款 |

### 3.4 时间同步机制

系统采用双向时间同步机制，确保链上时间与系统时间误差≤1分钟：

1. **后端同步**：每30秒自动同步链上时间与系统时间
   - 使用 `evm_mine` 出块（不带参数），Hardhat会自动使用 `max(上一区块+1, 系统时间)` 作为新区块时间戳
   - 链上时间落后系统时间超过30秒时自动调用 `evm_mine` 追上
   - 链上时间超前系统时间：Hardhat不支持回拨，等待系统时间自然追上
   
2. **交易前同步**：每次执行合约交易前自动检查时间同步，确保截止时间验证通过

3. **前端适配**：创建项目时使用链上时间计算截止时间
   - 获取链上当前时间戳
   - 计算用户期望的剩余时间（秒）
   - 实际截止时间 = 链上时间 + 用户期望时长
   - 确保截止时间至少比链上时间晚1分钟

4. **时区显示**：所有时间显示使用UTC+8（北京时间），使用链上时间计算剩余时间

> **注意**：Hardhat测试链时间只能向前推进，无法向后调整。系统优先使用 `evm_mine` 而非 `evm_increaseTime`，避免时间超前问题。系统误差≤1分钟。

## 4. 核心功能模块

### 4.1 智能合约层 (Crowdfunding.sol)

#### 4.1.1 数据结构

```solidity
// 项目状态枚举
enum ProjectStatus {
    Active,      // 活跃中
    Successful,  // 成功
    Failed       // 失败
}

// 阶段点结构 - 作业要求的里程碑（达到目标%发放资金）
struct Stage {
    string description;              // 阶段点描述
    uint256 completionThresholdBps;  // 达成阈值（基点，0-10000）
    uint256 releaseBps;              // 释放比例（基点）
    bool completed;                  // 是否已完成
    bool released;                   // 是否已释放
}

// 里程碑结构 - 发起者承诺的回报
struct Reward {
    uint256 fundingThreshold;    // 达标金额（wei）
    string promiseText;          // 发起者承诺内容
    uint256 expectedMonth;       // 预计实现时间（月）
    uint256 backersCount;        // 支持者数量
}

// 月度支持结构
struct MonthlySupport {
    uint256 monthlyAmount;      // 每月支持金额（wei）
    uint256 nextBillingTime;     // 下次扣款时间
    bool active;                 // 是否激活
}

// 项目结构
struct Project {
    uint256 id;              // 项目ID
    address creator;         // 创建者地址
    string name;             // 项目名称
    string description;      // 项目描述
    uint256 goal;            // 目标金额（wei）
    uint256 deadline;        // 截止时间
    uint256 pledged;         // 已筹金额
    uint256 balance;         // 合约余额
    uint256 totalWithdrawn;  // 总提现金额
    uint256 totalRefunded;   // 总退款金额
    ProjectStatus status;    // 项目状态
    bool ended;              // 是否结束
    bool creatorWithdrawn;   // 创建者是否已提现
    bool hasStages;          // 是否开启阶段点
    bool hasMonthlySupport;  // 是否开启月度支持
    uint256 donorCount;      // 捐赠者数量
}
```

#### 4.1.2 核心函数

**1. 创建项目 (createProject)**

```solidity
function createProject(
    string calldata name,
    string calldata description,
    uint256 goal,
    uint256 deadline,
    bool hasStages,                              // 是否开启阶段点
    string[] calldata stageDescriptions,           // 阶段点描述
    uint256[] calldata stageThresholdsBps,        // 阶段点达标比例（0-10000）
    uint256[] calldata stageReleaseBps,           // 阶段点释放比例
    bool hasMonthlySupport,                      // 是否开启月度支持
    uint256[] calldata rewardThresholds,           // 里程碑达标线
    string[] calldata rewardPromises,              // 里程碑承诺内容
    uint256[] calldata rewardMonths               // 里程碑预计实现时间（月）
) external returns (uint256)
```

**算法逻辑：**
- 验证目标金额 > 0
- 验证截止时间 > 当前时间
- 验证阶段点数组长度一致（如果启用）
- 验证里程碑数组长度一致（如果启用）
- 验证阶段点达标线不得超过目标
- 分配项目ID（自增）
- 初始化项目数据结构
- 创建阶段点记录和里程碑记录
- 触发 ProjectCreated 事件

**2. 捐赠 (donate)**

```solidity
function donate(uint256 projectId) external payable
```

**算法逻辑：**
- 验证项目存在且状态为 Active
- 验证项目未过期
- 验证捐赠金额 > 0
- 如果是新捐赠者，添加到捐赠者列表
- 前10名捐赠者标记为早期支持者
- 更新捐赠记录和项目余额
- 检测是否达到目标：无里程碑项目达标后立即结束，溢出金额退还最后捐赠者
- 触发 DonationReceived 事件

**3. 无里程碑项目达标立即结束 (_finishProjectImmediately)**

```solidity
function _finishProjectImmediately(uint256 projectId, address lastDonor) private
```

**算法逻辑：**
- 标记项目为已结束，状态设为 Successful
- 计算溢出金额（pledged - goal）
- 将溢出金额退还给最后捐赠者
- 调用 `_settleSuccessfulProject` 全额打款给发起者
- 触发 ProjectFinished 事件

**4. 成功项目结算 (_settleSuccessfulProject)**

```solidity
function _settleSuccessfulProject(uint256 projectId) private
```

**算法逻辑：**
- 所有阶段点自动标记为 completed=true、released=true
- 一次性全额打款给发起者（无论是否有阶段点）
- 设置 creatorWithdrawn=true
- 触发 CreatorWithdrawal 事件

**5. 自动结束到期项目 (checkAndFinishExpired)**

```solidity
function checkAndFinishExpired() external
```

**算法逻辑：**
- 遍历所有项目
- 检查项目是否未结束且已过期
- 如果已筹金额 ≥ 目标金额：状态设为 Successful，调用 `_settleSuccessfulProject`
- 如果已筹金额 < 目标金额：状态设为 Failed，按比例计算退款（阶段点资金保留）
- 标记项目为已结束
- 触发 ProjectFinished 事件

**6. 结束项目 (finishProject)**

```solidity
function finishProject(uint256 projectId) external
```

**算法逻辑：**
- 验证项目未结束
- 验证截止时间已到达
- 比较已筹金额与目标金额：
  - 已筹 ≥ 目标：状态设为 Successful，调用 `_settleSuccessfulProject`
  - 已筹 < 目标：状态设为 Failed，计算阶段点保留金额
- 标记项目为已结束
- 触发 ProjectFinished 事件

**7. 创建者提现 (creatorWithdraw)**

```solidity
function creatorWithdraw(uint256 projectId) external
```

**算法逻辑：**
- 验证项目存在
- 验证调用者是项目创建者
- 验证项目状态为 Successful 或 Failed
- 验证创建者未提现
- 将全部余额转账给创建者
- 更新提现记录
- 触发 CreatorWithdrawal 事件

**8. 退款 (refund)**

```solidity
function refund(uint256 projectId) external
```

**算法逻辑：**
- 验证项目存在
- 验证项目状态为 Failed
- 验证捐赠者有捐赠记录且未退款
- 计算可退款比例：`refundBps = (pledged - totalWithdrawn) / pledged`
- 计算实际退款金额：`refundAmount = donationAmount * refundBps`
- 验证退款金额 > 0 且余额充足
- 返还退款金额
- 标记已退款（保留原始捐赠记录）
- 更新退款记录
- 触发 DonationRefunded 事件

**9. 释放阶段点资金 (releaseStageFunds)**

```solidity
function releaseStageFunds(uint256 projectId, uint256 stageIndex) external
```

**算法逻辑：**
- 验证项目存在且状态为 Active 或 Failed（成功项目禁止调用，已自动全额打款）
- 验证调用者是项目创建者
- 验证阶段点已完成但未释放
- 计算释放金额：`amount = goal * releaseBps / BPS`
- 验证释放金额 > 0 且余额充足
- 转账给创建者
- 标记阶段点为已释放
- 触发 StageFundsReleased 事件

**10. 开始月度支持 (startMonthlySupport)**

```solidity
function startMonthlySupport(uint256 projectId, uint256 monthlyAmount) external
```

**算法逻辑：**
- 验证项目存在且状态为 Active
- 验证项目已开启月度支持
- 验证月度金额 > 0
- 验证捐赠者尚未开启月度支持
- 计算下次扣款时间（下月1日0:0:00）
- 创建月度支持记录
- 触发 MonthlySupportStarted 事件

**11. 执行月度扣款 (chargeMonthlySupport)**

```solidity
function chargeMonthlySupport(uint256 projectId, address supporter) external
```

**算法逻辑：**
- 验证项目存在且状态为 Active
- 验证月度支持已激活
- 验证当前时间 ≥ 下次扣款时间
- 验证捐赠金额 > 0
- 调用 donate 函数执行捐赠
- 更新下次扣款时间
- 触发 MonthlySupportCharged 事件

### 4.2 后端服务层 (Python/FastAPI)

#### 4.2.1 认证模块 (auth.py)

**数据结构：**

```python
@dataclass
class User:
    username: str              # 用户名
    password_hash: str         # 密码哈希
    address: str               # 区块链地址
    nickname: str              # 昵称
    internal_balance: float    # 内置余额（用于众筹）
    external_balance: float    # 外置钱包余额
```

**核心功能：**

1. **用户注册**
   - 创建新的区块链账户
   - 生成随机外置钱包余额（50-500 ETH）
   - 从测试链默认账户转账100 ETH用于支付gas
   - 存储用户信息

2. **用户登录**
   - 验证用户名和密码
   - 生成JWT token
   - 返回token和用户信息

3. **双钱包操作**
   - **充值 (topup)**：从外置钱包转入内置余额
   - **提现 (withdraw)**：从内置余额转出到外置钱包
   - **退款处理**：优先返还到内置余额，不足时从外置钱包扣除

#### 4.2.2 区块链服务模块 (service.py)

**核心类：CrowdfundingService**

1. **初始化**
   - 通过 HTTP RPC 连接到 Hardhat 本地链
   - 加载智能合约ABI和地址
   - 初始化Web3实例

2. **项目操作**
   - `create_project()`: 调用合约创建项目（12个参数）
   - `get_project()`: 获取项目详情（含阶段点、里程碑、捐赠者列表）
   - `list_projects()`: 获取所有项目列表

3. **捐赠操作**

#### 5.1.1 阶段点数据结构

    uint256 completionThresholdBps;  // 达成阈值（基点，1/10000）

    bool released;                   // 是否已释放

#### 5.1.2 阶段点判断算法

```javascript
// 前端判断是否显示阶段点操作按钮
const stageActions = isCreator && project.stages && project.stages.length > 0 
  ? project.stages.map(s => {
      // 计算达成阈值
      const threshold = (project.goalEth * s.completionThresholdBps) / 10000;
      
      // 判断是否达到阈值（使用原始pledgedEth）
      const reached = parseFloat(project.pledgedEth) >= parseFloat(threshold);
      
      // 根据状态显示不同按钮
      if (s.completed && !s.released) {
        // 已完成但未释放 → 显示释放资金按钮
        return `<button onclick="releaseStageFunds(${project.id}, ${s.index})">
          释放阶段点 #${s.index + 1} 资金
        </button>`;
      } else if (!s.completed && reached) {
        // 未完成但已达到阈值 → 显示完成按钮
        return `<button onclick="completeStage(${project.id}, ${s.index})">
          完成阶段点 #${s.index + 1}
        </button>`;
      }
      return "";
    }).join("") 
  : "";
```

// 计算释放金额（基于目标金额，而非筹集金额）

#### 5.4.1 自动结束逻辑

function checkAndFinishExpired() external {
    for (uint256 i = 1; i <= nextProjectId; i++) {
        if (_projectExists(i)) {
            Project storage project = projects[i];
            if (!project.ended && block.timestamp >= project.deadline) {
                // 自动结束项目
                project.ended = true;
                
                if (project.pledged >= project.goal) {
                    project.status = ProjectStatus.Successful;
                    _settleSuccessfulProject(i); // 一次性全额打款
                } else {

                    if (project.hasStages) {
                        _calculateProportionalRelease(i); // 计算阶段点保留
                    }
                }
                
                emit ProjectFinished(i, project.status, project.pledged, project.goal);
            }
        }
    }
}

#### 5.4.2 无里程碑项目达标立即结束

```solidity
function _finishProjectImmediately(uint256 projectId, address lastDonor) private {
    Project storage project = projects[projectId];
    
    project.ended = true;
    project.status = ProjectStatus.Successful;

    // 计算溢出金额
    uint256 excessAmount = project.pledged - project.goal;

    // 如果有溢出金额，退还给触发完成的捐赠者
    if (excessAmount > 0 && project.balance >= excessAmount) {
        project.balance -= excessAmount;
        project.totalRefunded += excessAmount;
        
        (bool success, ) = payable(lastDonor).call{value: excessAmount}("");
        require(success, "excess refund failed");
        
        emit DonationRefunded(projectId, lastDonor, excessAmount);
    }

    // 调用结算函数，一次性全额打款给发起者
    _settleSuccessfulProject(projectId);

    emit ProjectFinished(projectId, project.status, project.pledged, project.goal);
}
```

#### 5.4.3 成功项目结算

```solidity
function _settleSuccessfulProject(uint256 projectId) private {
    Project storage project = projects[projectId];
    
    // 所有阶段点自动标记为已完成、已释放
    if (project.hasStages) {
        for (uint256 i = 0; i < projectStages[projectId].length; i++) {
            Stage storage stage = projectStages[projectId][i];
            stage.completed = true;
            stage.released = true;
        }
    }
    
    // 一次性全额打款给发起者
    uint256 amount = project.balance;
    if (amount > 0) {
        project.balance = 0;
        project.creatorWithdrawn = true;
        project.totalWithdrawn += amount;
        
        (bool success, ) = payable(project.creator).call{value: amount}("");
        require(success, "auto withdraw failed");
        
        emit CreatorWithdrawal(projectId, project.creator, amount);
    }
}
```

#### 5.4.4 比例退款逻辑

```solidity
function refund(uint256 projectId) external projectExists(projectId) {
    Project storage project = projects[projectId];
    require(project.status == ProjectStatus.Failed, "project not failed");

    uint256 donationAmount = donations[projectId][msg.sender];
    require(donationAmount > 0, "nothing to refund");
    require(!donatedRefunded[projectId][msg.sender], "already refunded");

    // 计算可退款比例
    uint256 totalPledged = project.pledged;
    uint256 totalWithdrawn = project.totalWithdrawn;
    
    uint256 refundBps = 0;
    if (totalPledged > totalWithdrawn) {
        refundBps = ((totalPledged - totalWithdrawn) * BPS) / totalPledged;
    }
    
    // 计算实际退款金额
    uint256 refundAmount = (donationAmount * refundBps) / BPS;
    require(refundAmount > 0, "nothing to refund after stage release");
    require(project.balance >= refundAmount, "insufficient project balance");

    // 保留原始捐赠记录，标记已退款
    donatedRefunded[projectId][msg.sender] = true;
    project.totalRefunded += refundAmount;
    project.balance -= refundAmount;

    (bool success, ) = payable(msg.sender).call{value: refundAmount}("");
    require(success, "refund failed");

    emit DonationRefunded(projectId, msg.sender, refundAmount);
}
```

**示例：**
- 项目目标：10 ETH
- 筹集金额：8 ETH（80%）
- 阶段点1：达到60% → 释放20%目标金额 = 2 ETH
- 阶段点2：达到80% → 释放20%目标金额 = 2 ETH
- 已释放总金额：4 ETH
- 剩余资金：8 - 4 = 4 ETH
- 退款比例：4/8 = 50%
- 捐赠者捐赠8 ETH → 可退款 8 × 50% = 4 ETH

## 10. 系统流程图

### 10.1 用户注册流程

```
用户填写注册信息
        ↓
后端验证用户名唯一性
        ↓
创建区块链账户 (Web3.py)
        ↓
生成随机外置钱包余额 (50-500 ETH)
        ↓
从测试链转账100 ETH (用于支付gas)
        ↓
密码哈希存储
        ↓
返回用户信息
```

### 10.2 捐赠流程

```
用户选择项目
        ↓
输入捐赠金额
        ↓
选择是否开启月度支持
        ↓
验证登录状态
        ↓
检查内置余额
        ↓
调用后端API (/api/projects/{id}/donate)
        ↓
后端验证token
        ↓
后端扣除内置余额
        ↓
调用智能合约 donate()
        ↓
合约验证项目状态
        ↓
合约更新捐赠记录
        ↓
合约检测是否达标（无里程碑项目立即结束）
        ↓
合约触发 DonationReceived 事件
        ↓
返回交易哈希
        ↓
前端更新项目列表
```

### 10.3 阶段点资金释放流程

```
筹款达到阶段点阈值
        ↓
前端显示"完成阶段点"按钮
        ↓
创建者点击完成
        ↓
调用智能合约 completeStage()
        ↓
合约验证创建者身份
        ↓
合约标记阶段点完成
        ↓
前端显示"释放资金"按钮
        ↓
创建者点击释放
        ↓
调用智能合约 releaseStageFunds()
        ↓
合约验证阈值达成
        ↓
合约计算释放金额（goal × releaseBps / BPS）
        ↓
合约转账给创建者
        ↓
合约标记阶段点已释放
        ↓
资金转入创建者外置钱包
```

### 10.4 无里程碑项目达标立即结束流程

```
捐赠金额使项目达到目标
        ↓
合约检测无里程碑
        ↓
立即设置状态为Successful
        ↓
计算溢出金额（pledged - goal）
        ↓
溢出金额退还给最后捐赠者
        ↓
所有阶段点自动标记完成/释放
        ↓
一次性全额打款给发起者
        ↓
触发ProjectFinished事件
```

### 10.5 自动结束与退款流程

```
项目截止时间到达
        ↓
系统自动调用 checkAndFinishExpired()
        ↓
判断筹集金额与目标金额：
        ↓
    ┌─────────────────┬─────────────────┐
    ↓                 ↓                 ↓
≥ 目标金额         < 目标金额        项目进行中
    ↓                 ↓                 ↓
状态=Successful    状态=Failed       继续捐赠
    ↓                 ↓
自动全额打款      捐赠者可申请退款
    ↓                 ↓
所有阶段点默认完成  按比例退款：
                      refundAmount = 
                      donation × 
                      (pledged - withdrawn) / pledged
```

### 10.6 月度支持流程

```
用户选择项目
        ↓
项目已开启月度支持
        ↓
用户输入每月支持金额
        ↓
调用智能合约 startMonthlySupport()
        ↓
合约创建月度支持记录
        ↓
计算下次扣款时间（下月1日0:0:00）
        ↓
每月1日0:0:00自动执行
        ↓
调用 chargeMonthlySupport()
        ↓
合约验证支持已激活
        ↓
合约执行 donate()
        ↓
更新下次扣款时间
```

## 11. 总结

### 11.1 项目亮点

1. **双机制设计创新**
   - **阶段点**：基于目标比例的资金释放机制
   - **里程碑**：发起者承诺的回报展示机制
   - 两者独立运作，互不干扰

2. **无里程碑项目自动结束**
   - 达到目标后立即结束，溢出金额退还最后捐赠者
   - 资金自动到账，无需手动提取

3. **成功项目全额打款**
   - 所有阶段点默认已完成
   - 一次性全额打款给发起者
   - 简化操作流程

4. **月度支持功能**
   - 支持按月自动扣款的长期捐赠模式
   - 每月1日0:0:00自动执行
   - 提高捐赠的便利性和持续性

5. **精确时间同步**
   - 链上时间与系统时间同步，误差≤1分钟
   - 每30秒自动同步
   - 前端自动调整截止时间

6. **完整的用户认证**
   - JWT token认证
   - 密码哈希存储
   - 区块链账户绑定

7. **安全可靠**
   - 智能合约访问控制
   - 余额验证机制
   - 重入攻击防护
   - 防重复退款

### 11.2 技术难点与解决方案

| 难点 | 解决方案 |
|------|----------|
| 新用户无测试链余额 | 注册时从默认账户转账100 ETH |
| 捐赠需要支付gas | 使用内置余额记录捐赠，外置钱包支付gas |
| 阶段点判断复杂 | 前端实时计算阈值，后端合约验证 |
| 退款分配逻辑 | 按比例退款，已释放阶段点资金不退 |
| 用户身份管理 | JWT token + 区块链地址双重验证 |
| 时间同步问题 | 双向同步机制，前端自动调整截止时间 |
| 自动结束项目 | 添加 checkAndFinishExpired() 函数 |
| 月度支持时间计算 | 自定义函数计算下月1日0:0:00 |
| 无里程碑项目达标处理 | _finishProjectImmediately() 立即结束并退款 |
| 阶段点字段访问错误 | 修复 project.rewardCount 为 projectRewards[projectId].length |
| 退款后捐赠记录丢失 | 使用 donatedRefunded 映射标记，保留原始记录 |
